JavaScript form validation

Task:

Release form validation. Form fields must be validated while typing (but not showing errors before the user has entered something).

The form cannot be submitted if there are errors.

If errors are shown, the form submit button should be inactive (the inactive button code is commented out).

Required fields are marked with asterisks.

The error fields should be highlighted properly (which can be seen by uncommenting the code on the bootstrap page).

All regulars must be written by yourself.

Possible error situations

For each, create and display an explanatory message.

It should be clear from the message what the problem is. Do not use variants ("something is wrong").

  1. Required field not filled
  2. Email error
  3. email is already taken (check the static list of emails stored globally in the usedEmails variable)
    [author@mail.com, foo@mail.com, tester@mail.com]
  4. Password is too short (up to 5 characters)
  5. The password contains prohibited characters (allowed - Latin letters, numbers, underscore, minus)
  6. The international phone recording format is not consistent
  7. The checkbox "Agree with everything" is not checked
